## Алгоритм: BSP (Binary Space Partitioning)

### Шаги генерации
1. Карта 40x30 тайлов, заполнена стенами (0)
2. Рекурсивный BSP-сплит:
   - Делим пространство пополам (горизонтально или вертикально)
   - Минимальный лист = 7x7 тайлов
   - Глубина = 3-4 (даёт 5-8 комнат)
   - Направление сплита: если ширина > высоты*1.25 → вертикально, иначе горизонтально
3. В каждом листе — комната:
   - Размер: от 4x4 до (лист - 2) тайлов
   - Отступ минимум 1 тайл от края листа
   - Заливаем пол (1)
4. Коридоры между сёстрами по BSP-дереву:
   - Берём центры двух сестёр-листьев
   - L-образный коридор шириной 1-2 тайла
   - Заливаем пол (1)
5. Post-processing:
   - Flood-fill от стартовой комнаты — проверить что ВСЕ комнаты достижимы
   - Если нет — принудительный коридор к изолированной комнате
   - 1 дополнительный коридор между случайными комнатами (петля для нелинейности)

### Специальные комнаты
- Комната 0 (корень BSP) = стартовая, БЕЗ врагов
- Самая дальняя от старта (по BFS) = лестница вниз
- Опционально: комната с наибольшим расстоянием от других = сокровищница

### Параметры (из balance.json → dungeon)
- mapWidth: 40
- mapHeight: 30
- bspDepth: 3-4 (рандом на этаже)
- minLeafSize: 7
- roomPadding: 1
- corridorWidth: 1
- extraCorridors: 1

### Формат выхода DungeonMap
```ts
interface Room { x: number; y: number; w: number; h: number; type: 'start' | 'normal' | 'treasure' | 'stairs'; }
interface DungeonMap {
  tiles: number[][];      // 0=стена, 1=пол, 2=лестница
  rooms: Room[];
  playerStart: { x: number; y: number };
  stairPos: { x: number; y: number };
  width: number;
  height: number;
}
```

### Правила
- ВСЕ параметры из balance.json, не хардкодить
- Комнаты ВСЕГДА прямоугольные (тайлсет не поддерживает другое)
- Flood-fill ОБЯЗАТЕЛЕН после генерации
- Depth sorting пропсов в комнатах = sprite.y + sprite.height
- Враги спавнятся ТОЛЬКО в комнатах type='normal'
- Стартовая комната ВСЕГДА пустая
